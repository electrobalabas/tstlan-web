Модель данных
=============

Пользователи и сессии
---------------------

:class:`tstlan.auth.models.User` и :class:`tstlan.auth.models.Session` описывают
серверную аутентификацию. В cookie уходит только случайный token, CSRF-токен
возвращается API и должен отправляться в ``X-CSRF-Token`` на ``POST``, ``PUT``,
``PATCH`` и ``DELETE``.

Конфиги
-------

:class:`tstlan.configs.models.DeviceConfig` - сохранённый профиль подключения и
переменных прибора. :class:`tstlan.configs.models.ConfigShare` выдаёт точечный
доступ одному пользователю.

Эффективный доступ считается сервисом, а не хранится отдельным полем:

* ``owner`` - владелец конфига или admin.
* ``write`` - явный share с правом записи.
* ``read`` - явный read-share или публичный конфиг.

``shared`` - производная видимость: если у непубличного конфига есть shares, он
становится ``shared``, иначе ``private``. Пользователь явно выбирает только
публикацию ``public``; она доступна ``dev`` и ``admin``.

Payload конфига
---------------

:class:`tstlan.configs.schemas.ConfigPayload` состоит из подключения
(:class:`tstlan.configs.schemas.ConnectionSettings`) и списка переменных
(:class:`tstlan.configs.schemas.ConfigVar`). ``params`` сохраняет неизвестные
legacy-поля, чтобы конвертер :mod:`tstlan.tools.ini2yaml` не терял данные, для
которых ещё нет отдельной модели.

Переменные
----------

:class:`tstlan.configs.schemas.ConfigVar` используется в сохранённом конфиге.
Runtime-переменная :class:`tstlan.models.NetVar` дополнительно имеет режим:

* ``r`` - только чтение, сенсор.
* ``w`` - только запись, команда.
* ``rw`` - чтение и запись, настройка или управляющий регистр.

Целочисленные записи валидируются по диапазону C-типа. ``bool`` не принимается
даже для ``bit``: API ожидает числовое ``0`` или ``1``. ``f32`` и ``f64``
приводятся к ``float``.

Layout переменных
-----------------

Адрес переменной не хранится в БД. Он вычисляется из порядка ``variables`` через
:func:`tstlan.configs.schemas.variable_offsets`. Ту же функцию layout повторяет
frontend-редактор, поэтому порядок переменных - часть контракта конфига.

Frontend-контракт
-----------------

``web/tstlan-web/lib/api.ts`` зеркалит backend-схемы TypeScript-типами.
Изменяющие запросы принимают ``csrfToken`` явно и добавляют заголовок
``X-CSRF-Token``.

``web/tstlan-web/lib/configs.ts`` держит UI-валидацию формы: обязательные поля
для транспорта, диапазоны порта и GPIB, уникальность имён переменных и расчёт
смещений. Это ранняя проверка для пользователя, но окончательная валидация всё
равно остаётся на backend.
